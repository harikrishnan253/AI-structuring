## CRITICAL: XML MARKERS AND SEMANTIC CONTEXT

Many source documents contain XML-style markers that indicate the SEMANTIC PURPOSE of content. These markers tell you WHAT the content IS and WHY it's styled a certain way. You must understand these markers to classify correctly.

### Understanding XML Markers

XML markers in source documents are **editorial instructions** that indicate content purpose:

| Marker | Meaning | Why It Matters | Correct Style |
|--------|---------|----------------|---------------|
| `<NL>...</NL>` | Numbered List block | Content is a sequential list, often objectives | NL-* or OBJ-NL-* |
| `<BL>...</BL>` | Bullet List block | Content is an unordered list of items | BL-* |
| `<UNL>...</UNL>` | Unnumbered/Lettered List | Mnemonic or lettered list (A, B, C or T, I, C, L, S) | UL-* |
| `<BT>` | Box Title | Title text for a special box/callout | NBX-TTL, BX*-TTL |
| `<BN>` | Box Number | Box identifier like "BOX 4-1" | NBX-TTL (combined with title) |
| `<CL>` | Citation Line | Source attribution for figure/table | TSN, FIG-SRC |
| `<FN>` | Figure Number | Figure identifier | FIG-LEG |
| `<FT>` | Figure Title | Figure caption text | FIG-LEG |
| `<TN>` | Table Number | Table identifier | T1 |
| `<TT>` | Table Title | Table caption text | T1 |
| `<H1>`, `<H2>` | Heading markers | Heading level hints | H1, H2, etc. |

### Special Box/Callout Markers

These markers define **pedagogical elements** - special content boxes used in educational publishing:

| Marker | Purpose | Educational Function | Styles to Use |
|--------|---------|---------------------|---------------|
| `<NOTE>...</NOTE>` | Informational note | Highlights important supplementary information | NBX-* styles |
| `<CLINICAL PEARL>...</CLINICAL PEARL>` | Clinical insight | Practical tips from clinical experience (medical texts) | BX1-* styles |
| `<RED FLAG>...</RED FLAG>` | Warning/Alert | Critical safety information or warning signs | BX2-* or SP-* styles |
| `<BOX>...</BOX>` | General box | Numbered educational box with title | NBX-* styles |
| `<TIP>...</TIP>` | Helpful tip | Practical advice for readers | BX1-* styles |
| `<EXAMPLE>...</EXAMPLE>` | Worked example | Demonstration of concept application | BX1-* styles |

### Box Context Tracking

**CRITICAL**: When you encounter a box opening marker (e.g., `<NOTE>`, `<CLINICAL PEARL>`), ALL subsequent content until the closing marker belongs to that box and should use box-prefixed styles.

**Box Content Flow:**
```
<NOTE>                          ‚Üí PMI (marker only)
NOTE                            ‚Üí NBX-TTL (box type label)
<BT>Important Concept           ‚Üí NBX-TTL (box title)
First paragraph of note...      ‚Üí NBX-TXT-FIRST or BX1-TXT-FIRST
Second paragraph...             ‚Üí NBX-TXT or BX1-TXT
‚Ä¢ Bullet point                  ‚Üí NBX-BL-FIRST
‚Ä¢ Another bullet                ‚Üí NBX-BL-LAST
</NOTE>                         ‚Üí PMI (marker only)
```

**Box Type Mapping:**
- NOTE, TIP ‚Üí NBX-* (informational boxes)
- CLINICAL PEARL, EXAMPLE ‚Üí BX1-* (practical application boxes)
- RED FLAG, WARNING, ALERT ‚Üí BX2-* (warning/caution boxes)
- REFLECTION, DISCUSSION ‚Üí BX3-* (thinking/discussion boxes)
- CASE STUDY ‚Üí BX4-* (case study boxes)

### Mnemonic/Lettered Lists

Medical and educational texts frequently use **mnemonic devices** - memory aids using letter patterns:

**Pattern Recognition:**
```
T    Tone: Is the muscle tone...        ‚Üí UL-FIRST (or NBX-UL-FIRST if in box)
I    Interactiveness: Does the child... ‚Üí UL-MID
C    Consolability: Is the child...     ‚Üí UL-MID
L    Look/gaze: If the eyes...          ‚Üí UL-MID
S    Speech/cry: Is the child's...      ‚Üí UL-LAST
```

**Why UL-* (Unnumbered List)?**
- These are NOT regular bulleted lists (no bullets)
- These are NOT numbered lists (letters aren't sequential numbering)
- They are LETTERED lists where each letter is a mnemonic cue
- The `<UNL>` marker confirms this is an unnumbered/lettered list

**Detection Pattern:** Single uppercase letter + tab/spaces + description text
- `^[A-Z]\t` or `^[A-Z]  ` followed by text
- Usually forms a word or acronym (TICLS, SAMPLE, CIAMPEDS)

### Source/Citation Lines

The `<CL>` marker indicates a **citation or source line** - attribution for figures, tables, or adapted content:

**Why TSN (Table Source Note) or FIG-SRC?**
- Publishers need to track permissions and attributions
- Source lines must be formatted consistently for legal/copyright reasons
- They appear after figures/tables, not as body text

**Detection Patterns:**
- `<CL>` marker present
- Starts with: "Reproduced from", "Data from", "Adapted from", "Source:", "Courtesy of"
- Contains author names, publication info, dates

**Example:**
```
<CL>Data from Horeczko, T., Enriquez, B., McGrath, N. E....  ‚Üí TSN
<CL>Reproduced from Chemical Hazards Emergency Medical...    ‚Üí TSN
Source: American Academy of Pediatrics, 2023.                ‚Üí TSN
```

### Objectives Section

Learning objectives wrapped in `<NL>...</NL>` after an "Objectives" heading should use OBJ-* styles:

**Why OBJ-* styles?**
- Learning objectives are pedagogically distinct from regular lists
- They define what students should learn/achieve
- Publishers format them specially (often in boxes or highlighted areas)

**Pattern:**
```
<H1>Objectives                              ‚Üí H1
Upon completion of this chapter...          ‚Üí OBJ-TXT-FLUSH
<NL>Integrate Pediatric Assessment...       ‚Üí OBJ-NL-FIRST
Identify pediatric considerations...        ‚Üí OBJ-NL-MID
Recognize pediatric red flags.</NL>         ‚Üí OBJ-NL-LAST
```

## COMPLETE STYLE TAG REFERENCE (WK Book Template 1.1)

### CHAPTER OPENER
- **CW**: Chapter word ("CHAPTER" text only)
- **CN**: Chapter number ("CHAPTER 2", "Chapter 8", or just "2")
- **CT**: Chapter title (main title following CN)
- **CST**: Chapter subtitle
- **CAU**: Chapter author names
- **CHAP**: Word "Chapter" on opener
- **COQ**: Chapter opener quote
- **COQA**: Chapter opener quote author

### PART/UNIT/SECTION OPENERS
- **PART**: Word "Part" on opener
- **PN**: Part number
- **PT**: Part title
- **PST**: Part subtitle
- **PAU**: Part author names
- **UNIT**: Word "Unit" on opener
- **UN**: Unit number
- **UT**: Unit title
- **UST**: Unit subtitle
- **SECTION**: Word "Section" on opener
- **SN**: Section number
- **ST**: Section title
- **SST**: Section subtitle

### HEADINGS (H1-H6 Hierarchy)
- **H1**: Level 1 head (primary sections)
- **H2**: Level 2 head (subsections)
- **H3**: Level 3 head (sub-subsections)
- **H4**: Level 4 head (run-in or minor heads)
- **H5**: Level 5 head (detailed sub-points, often run-in)
- **H6**: Level 6 head
- **H1A**: Chapter outline head ("Chapter Outline")
- **H2 after H1**: Level 2 head immediately following H1
- **H3 after H2**: Level 3 head immediately following H2
- **SP1**: Special head, first level
- **SP2**: Special head, second level
- **SPH2**: Special purpose heading level 2
- **PT-H2**: Part/Section heading within chapter (e.g., "Section 11.1: Wellness & Stress")

### BODY TEXT
- **TXT**: Chapter text (with first-line indent)
- **TXT-FLUSH**: Chapter text (no first line indent - use after headings)
- **TXT-DC**: Chapter text with drop cap (opening paragraph)
- **TXT-SPACE ABOVE**: Chapter text with one line of space above
- **TXT-AU**: Author initials at end of section
- **INTRO**: Italic lead-in text
- **APTX**: Additional/supplementary paragraph text
- **TX**: Text (alternate style name)
- **TXL**: Text with lead-in (alternate style name)

### BULLETED LISTS (Position Required: FIRST/MID/LAST)
- **BL-FIRST**: Bulleted list, first instance
- **BL-MID**: Bulleted list, middle instances
- **BL-LAST**: Bulleted list, last instance
- **BL-TXT**: Text continuation within bulleted list (elaborates on preceding bullet)
- **BL2-MID**: Bulleted list, level 2 subentry
- **BL3-MID**: Bulleted list, level 3 subentry
- **BL4-MID**: Bulleted list, level 4 subentry
- **BL5-MID**: Bulleted list, level 5 subentry
- **BL6-MID**: Bulleted list, level 6 subentry
- **BL**: Bulleted list (alternate - no position)
- **BLF**: Bulleted list first (alternate)
- **BLL**: Bulleted list last (alternate)
- **BLBL**: Bulleted list sub-bullet (alternate)
- **BLTXL**: Bulleted list with text lead-in

### NUMBERED LISTS (Position Required: FIRST/MID/LAST)
- **NL-FIRST**: Numbered list, first instance
- **NL-MID**: Numbered list, middle instances
- **NL-LAST**: Numbered list, last instance
- **NL-MID following L1**: Numbered list following list head
- **NL-TXT**: Text continuation within numbered list

## CRITICAL CLASSIFICATION RULES

### Rule 0: Document Front Matter / Metadata Handling
**IMPORTANT**: Many documents begin with metadata BEFORE the actual chapter content. Handle as follows:
- Lines containing "Book title:", "Chapter title:", "Section title:", "Author:", "Corresponding author:" ‚Üí **PMI** (metadata)
- Lines with email addresses (containing @) ‚Üí **PMI**
- Lines with "<metadata>" or "</metadata>" markers ‚Üí **PMI**
- Lines with "ABSTRACT:", "KEYWORDS:", "MESH TERMS:" ‚Üí **PMI**
- Empty lines before chapter number (CN) ‚Üí **PMI** or skip
- Once you see CN (Chapter Number), normal document tagging begins

### Rule 0a: Empty Paragraph Handling
- Empty paragraphs (blank lines, whitespace only) ‚Üí **PMI**
- Empty paragraphs act as visual separators and should not be tagged as TXT
- Do NOT count empty paragraphs when determining FIRST/MID/LAST list positions

### Rule 0b: XML Marker Hints in Text
Many documents contain XML-style markers that HINT at the correct tag:
- Text starting with "<H1>" ‚Üí **H1** (use H1 tag, marker is a hint)
- Text starting with "<H2>" ‚Üí **H2**
- Text starting with "<H3>" ‚Üí **H3**
- Text with "<TAB..." or "<TN>" ‚Üí **T1** (table title)
- Text with "<FIG..." or "<FN>" ‚Üí **FIG-LEG** (figure legend)
- Text with "<CL>" ‚Üí **TSN** (citation/source line)
- Text with "<BT>" ‚Üí **NBX-TTL** or **BX*-TTL** (box title)
- Text with "<BN>" ‚Üí **NBX-TTL** (box number + title)
- Text with "<NL>" or "</NL>" ‚Üí **NL-*** or **OBJ-NL-*** (numbered list)
- Text with "<UNL>" or "</UNL>" ‚Üí **UL-*** (unnumbered/lettered list)
- Text with "<BL>" or "</BL>" ‚Üí **BL-*** (bullet list)
- The marker in the text tells you what the element SHOULD be tagged as

### Rule 0c: Box Context Tracking (CRITICAL)
When you see a box opening marker, ALL content until closing marker uses box styles:

**Box Opening Markers** (set box context):
- `<NOTE>` ‚Üí Use NBX-* styles for content
- `<CLINICAL PEARL>` ‚Üí Use BX1-* styles for content
- `<RED FLAG>` ‚Üí Use BX2-* styles for content
- `<BOX>` ‚Üí Use NBX-* styles for content
- `<TIP>` ‚Üí Use BX1-* styles for content

**Inside Box Content Mapping:**
- Box type label (e.g., "NOTE", "CLINICAL PEARL") ‚Üí NBX-TTL or BX*-TTL
- Box title with `<BT>` marker ‚Üí NBX-TTL or BX*-TTL
- First paragraph after title ‚Üí NBX-TXT-FIRST or BX*-TXT-FIRST
- Subsequent paragraphs ‚Üí NBX-TXT or BX*-TXT
- Bullet lists ‚Üí NBX-BL-* or BX*-BL-*
- Numbered lists ‚Üí NBX-NL-* or BX*-NL-*
- Lettered/mnemonic lists ‚Üí NBX-UL-* or BX*-UL-*

**Box Closing Markers** (end box context):
- `</NOTE>`, `</CLINICAL PEARL>`, `</RED FLAG>`, `</BOX>` ‚Üí PMI

### Rule 0d: Mnemonic/Lettered List Detection
Medical texts use mnemonic lists (TICLS, SAMPLE, CIAMPEDS, etc.):

**Pattern:** Single letter + tab + description
```
T    Tone: description...     ‚Üí UL-FIRST (or NBX-UL-FIRST in box)
I    Interactiveness...       ‚Üí UL-MID
C    Consolability...         ‚Üí UL-MID
L    Look/gaze...             ‚Üí UL-MID
S    Speech/cry...            ‚Üí UL-LAST
```

**Detection:** `^[A-Z]\t` or `^[A-Z]  ` (letter + tab/spaces)
**Why UL-*:** These are lettered lists, not bulleted (BL-*) or numbered (NL-*)

### Rule 0e: Source/Citation Line Detection
Lines with `<CL>` marker or source attribution patterns:

**Patterns that indicate TSN (Table/Figure Source Note):**
- `<CL>` marker present
- Starts with "Reproduced from"
- Starts with "Data from"
- Starts with "Adapted from"
- Starts with "Source:"
- Starts with "Courtesy of"

**NEVER tag these as TXT** - they are attribution lines requiring TSN or FIG-SRC

### Rule 0f: Objectives Section Handling
Content after "Objectives" heading wrapped in `<NL>...</NL>`:

- "Objectives" or "Learning Objectives" heading ‚Üí H1
- Intro text ("Upon completion...") ‚Üí OBJ-TXT-FLUSH
- List items with `<NL>` marker ‚Üí OBJ-NL-FIRST
- Middle list items ‚Üí OBJ-NL-MID
- Last item (or with `</NL>`) ‚Üí OBJ-NL-LAST

### Rule 1: List Position Detection
- Single item only: Use FIRST only (no LAST)
- Two items: FIRST, LAST
- Three+ items: FIRST, MID (repeated), LAST

### Rule 2: Heading Hierarchy
- Must follow logical order (H1 ‚Üí H2 ‚Üí H3 ‚Üí H4)
- Never skip levels without context
- After H1, next head should be H2 (not H3)

### Rule 3: Container Context
- Inside "Box X.X" or box markers ‚Üí BX1-* or NBX-* tags
- Inside exercises ‚Üí EXER-* tags
- Inside appendix ‚Üí APX-* tags
- After "REFERENCES" ‚Üí REF-* tags
- End of chapter review ‚Üí EOC-*, RQ-*, QUES-* tags

### Rule 4: First Paragraph Rule
- First paragraph after ANY heading ‚Üí TXT-FLUSH (not TXT)
- Subsequent paragraphs ‚Üí TXT (with indent)

### Rule 5: Reference Detection
- Numbered entries (1., 2., etc.) after reference heading ‚Üí REF-N (not NL-*)
- Unnumbered references ‚Üí REF-U
- Bibliography style ‚Üí BIB

### Rule 6: Table Detection and Classification
Tables are detected by these patterns. Table content is often in actual Word tables, not paragraphs.

**Table Title Detection (T1):**
- Text with `<TAB#.#>Table #.#` pattern ‚Üí **T1** (e.g., "<TAB2.1>Table 2.1 - Title...")
- Text starting with "Table X.X" or "TABLE X.X" ‚Üí **T1**
- Text starting with "Table X.X:" with colon ‚Üí **T1**
- Text starting with "Table X.X ‚Äì" with dash ‚Üí **T1**
- Unnumbered table title (heading-like, before table) ‚Üí **UNT-T1**
- IMPORTANT: If text has BOTH marker AND title content, use **T1** (not PMI)

**Table Section Markers (PMI):**
- Lines with ONLY `<TAB#.#>` (no title text) ‚Üí **PMI**
- Lines with `<TABLES>` (section start) ‚Üí **PMI**
- Lines with `</>` (table placeholder end) ‚Üí **PMI**
- Empty Normal paragraphs between tables ‚Üí **PMI** or **Normal**

**Table Header Detection (T2, T2-C):**
- Column header text (first row of table) ‚Üí **T2**
- Centered/spanning column header ‚Üí **T2-C**
- Headers like "Component", "Function", "Description" ‚Üí **T2**
- Straddle headers spanning multiple columns ‚Üí **T2-C**

**Table Row Header Detection (T4):**
- First column content that identifies the row ‚Üí **T4**
- Category names in leftmost column ‚Üí **T4**
- Examples: "CAR T-Cells", "Macronutrients", "Low-Impact Activities"

**Table Body Cell Detection (T, T5, T6):**
- Regular data cells ‚Üí **T**
- Specific data values/measurements ‚Üí **T5**
- Alternative data type ‚Üí **T6**

**Table List Content (TBL-*, TNL-*):**
- Bulleted list inside table cell ‚Üí **TBL-FIRST**, **TBL-MID**, **TBL-LAST**
- Numbered list inside table cell ‚Üí **TNL-FIRST**, **TNL-MID**, **TNL-LAST**
- Sub-bullets in table ‚Üí **TBL2-MID**, **TBL3-MID**

**Table Footnote Detection (TFN):**
- Abbreviation definitions: "BID, twice a day; TIC, three times..." ‚Üí **TFN**
- Text starting with "a", "b", "*", "‚Ä†" (superscript references) ‚Üí **TFN**
- Text starting with "Note:" or "Notes:" after table ‚Üí **TFN**
- Text with "aExcludes...", "bIncludes..." ‚Üí **TFN**
- Abbreviation lists: "BMI, body mass index; LVH, left ventricular..." ‚Üí **TFN**
- "Adapted from..." or "Adapted in part from..." ‚Üí **TFN** (or TSN)

**Table Source Note Detection (TSN):**
- Text starting with "Source: " ‚Üí **TSN**
- Text starting with "From " followed by citation ‚Üí **TSN**
- Text starting with "Data from " ‚Üí **TSN**
- Usually appears AFTER all TFN entries

**Table Sequence Pattern:**
Typical table sequence in documents:
1. PMI (`<TABLES>`) - optional section marker
2. T1 (`<TAB2.1>Table 2.1: Title...`) - table title
3. PMI (`</>`) - placeholder end marker
4. Normal (empty) - spacing
5. T1 (next table) or body text continues

### Rule 7: PMI Markers (Print Media Instructions)
- Lines containing ONLY markers like <TAB2.1>, <DFIG5.1>, <FIG71.1> ‚Üí PMI
- Lines with <INSERT>, <TABLES>, </>, <BL>, </BL>, <START BOX>, <END BOX> ‚Üí PMI
- If marker is embedded in content (e.g., "<TAB5.1>Table 5.1: Title...") ‚Üí Use content style (T1)
- These are placeholder instructions for production, not actual content

### Rule 8: Nested Lists
- Level 2 bullets (indented, ‚óã, o) ‚Üí BL2-MID
- Level 3 bullets ‚Üí BL3-MID
- Continue pattern for deeper nesting
- Nested bullets under numbered lists (1. item then o sub-bullet) ‚Üí BL2-MID

### Rule 9: Table Context Awareness
- After T1 (table title), expect: TFN, TSN, or return to body text
- TFN entries often appear in sequence (multiple footnotes)
- TSN usually appears as last element of table section
- Normal/blank paragraphs between tables are common

### Rule 9a: Table Cell Content Classification
**CRITICAL**: Content from Word table cells needs special classification based on position.
When you see markers like `<TABLE1,HEADER_ROW>` or `<TABLE1,FIRST_COL>` or `<TABLE1,R2C1>`:

**Header Row Content (HEADER_ROW)**:
- Column titles like "Component", "Function", "Description" ‚Üí **T2**
- Spanning/straddle headers ‚Üí **T2-C**

**First Column Content (FIRST_COL)**:
- Row labels/identifiers ‚Üí **T4**
- Examples: "CAR T-Cells", "Initial lesion outbreak", "Emotional"

**Body Cell Content (R#C#)**:
- Regular data cells ‚Üí **T**
- Cells with measurements, dosages ‚Üí **T5**
- Bulleted content in cells ‚Üí **TBL-MID**

**DO NOT** classify table cell content as TXT or TXT-FLUSH!
Table cells should use T, T2, T4, T5, T6, TBL-*, or TNL-* tags.

### Rule 10: Pull Quotes and Pull Boxes
- Text starting with "Pull Quote:" ‚Üí QUO
- Text starting with "Pull Box:" ‚Üí BX2-TXT-FLUSH (first) or BX2-TXT
- Pull boxes contain reflection/analysis content, not examples
- Pull quotes are standalone inspirational/key messages

### Rule 11: Box Type Detection
- **BX1-* (Type 1 Box - Examples)**: Contains "Example:", practical scenarios, case comparisons
- **BX2-* (Type 2 Box - Reflection/Pull Box)**: Contains "Pull Box:", theoretical insights, analysis
- **BX3-* (Type 3 Box - Discussion)**: Contains discussion prompts, critical thinking
- **BX4-* (Type 4 Box - Case Study)**: Contains "Case Study:", detailed scenarios
- Box content after title: First paragraph is typically BX*-TXT or BX*-TXT-FLUSH

### Rule 12: List Continuation Text (NL-TXT, BL-TXT)
- Text that continues/explains a list item without being a new item ‚Üí NL-TXT or BL-TXT
- Usually appears after a numbered list item, providing elaboration
- Does NOT have a number or bullet prefix
- Example: After "1. Character and Integrity:", the next paragraph elaborating ‚Üí NL-TXT

### Rule 13: Chapter Structure for Business/Educational Documents
- Major topic sections (after chapter opener) ‚Üí H1
- Subsections within H1 ‚Üí H2
- Sub-subsections or specific topics ‚Üí H3
- IMPORTANT: In business textbooks, main named sections like "Defining Persuasion" are H1, not H2
- Subheads like "Ethos: The Appeal to Credibility" under a main section ‚Üí H2

### Rule 14: End of Chapter Structure
- "Chapter Summary" or "Summary" ‚Üí H1 (not EOC-H1 unless in EOC section)
- "Suggested Readings", "Works Cited", "References" ‚Üí EOC-H1
- Bulleted/numbered references after EOC-H1 ‚Üí EOC-REF
- "What Can You Do Tomorrow?" type application sections ‚Üí H1

### Rule 15: AIDA/PAS Framework Lists
- Framework steps (Attention:, Interest:, Desire:, Action:) ‚Üí UL-FIRST/MID/LAST
- Example text after each step ‚Üí UL-MID (part of same list flow)
- These are NOT separate TXT paragraphs
